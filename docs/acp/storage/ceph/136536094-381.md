# 3.8.1

osd的pod处于init:crashloopbackoff状态 mon的pod有四个其中一个pending /var/lib/rook/rook-ceph/log目录占用62G磁盘空间

## Cause
- ceph mon异常导致osd启动异常
- coredump日志导致磁盘空间占满

## Resolution
- 删除多余的mon deployment并重启rook-ceph-operator
- 在cephcluster配置中添加字段防止coredump刷日志

## [workaround]
