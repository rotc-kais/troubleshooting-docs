# 3.12.1

应用访问nacos内部域名报错无法解析 通过coredns svc ip无法解析 通过coredns pod ip可以解析

## Cause
- coredns configmap中配置的默认端口为53，实际3.12版本应使用1053端口

## Resolution
- 修改coredns configmap的默认端口配置为1053
