# 中信银行

underlay pod无法访问overlay的coredns 同一主机上部分pod能通部分不通 删除异常pod后在其他节点重建仍不通

## Cause
- podman容器使用kube-ovn-pinger的IP地址导致IP冲突
- ovs-vswitchd存在未释放的数据结构/流表打满
- coredns回包未到达物理网卡

## Resolution
- 重启问题节点的ovs-ovn组件
- 按照方案卸载podman容器

## [workaround]
- 临时重启业务节点的ovs-ovn组件
