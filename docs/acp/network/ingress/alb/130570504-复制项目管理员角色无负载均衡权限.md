# 复制项目管理员角色，无负载均衡权限

复制项目管理员角色、命名空间管理员角色无负载均衡权限

## Cause
- 复制的角色缺少configmaps资源权限配置

## Resolution
- 在global集群的infrastructure-loadbalance资源中，spec.rules添加：
- apiGroup: ''
  bindCluster: unlimit
  bindNamespacePart: system
  bindScope: namespace
  resources:
    - configmaps
- 在metadata.annotations添加：skip-sync: 'true'
- 重启global的auth-controller2
