# 检查脚本没检查出来的ovn db 异常

cni 返回 500 controller 显示 db 操作失败 db log 显示存储空间满导致异常

## Cause
- 两台 master 存储空间满导致 ovn db 异常

## Resolution
- 检查 db log
- 手动操作 ko 插件重启
- 使用 db_check 脚本恢复

## [workaround]

- Environment: Kubernetes, CNI 1.9
- ovn db

- db_check 脚本
