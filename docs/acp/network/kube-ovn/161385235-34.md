# 3.4

coredns和大量容器无法启动 连接kubernetes超时(443) curl测试正常但pod启动失败

## Cause
- ovn checksum功能导致overlay网络异常
- 内核/网卡驱动问题导致udp checksum异常

## Resolution
- 在每个ovs-ovn-xxxx的pod中执行命令：ovs-vsctl set open . external-ids:ovn-encap-csum=false
