# 电科院 kube

master02节点上的所有Pod无法与其他节点Pod通信 从master02节点Pod发出的报文在物理网卡抓不到 OVS流表缺少output to kernel tunnel的日志

## Cause
- mac_binding表中存在无效或冲突的datapath_id
- ovn-controller状态异常导致流表转换失败

## Resolution
- 重启集群所有主机节点

## [workaround]
- 手动删除无效的mac_binding表项
