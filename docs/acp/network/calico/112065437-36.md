# 3.6

添加节点失败，报错'无法访问到节点'

## Cause
- 集群计算节点数已达上限（cluster CIDR为/22时最大支持4个节点）

## Resolution
- 修改maxNodePodNum为128
- 修改nodeCIDRMaskSize为25
- 更新所有业务集群节点的kubelet配置maxPods为128并重启kubelet
