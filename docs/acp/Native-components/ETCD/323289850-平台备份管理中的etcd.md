# 平台备份管理中的etcd

etcd-backup-default 备份策略无法删除 删除后自动恢复 备份失败时不断创建Pod导致集群损坏

## Cause
- 3.14.2版本默认策略被设计为不可删除
- 节点异常触发重复备份操作

## Resolution
- 将etcd备份方式改为手动触发
- 关闭自动定时备份功能

## [workaround]
- 通过后台直接备份etcd数据
