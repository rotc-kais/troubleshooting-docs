# 多网卡自建vip问题

无法通过vip进行访问 keepalived自己节点的另一个ip发vrrp导致leader被自己抢占

## Cause
- 未指定vrrp_check_unicast_src配置
- peer配置中包含自己的ip

## Resolution
- 在global master节点添加vrrp_check_unicast_src配置
- 删除keepalived配置中的peer配置
- 重建节点上keepalived容器
