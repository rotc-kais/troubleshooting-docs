# 3.8

添加第二个磁盘设备导致第一个设备不可用 清理异常磁盘后第一块磁盘仍显示异常

## Cause
- lvmdconfig配置中异常节点的状态字段未处于ready状态

## Resolution
- kubectl get cm -n nativestor-system
- kubectl edit cm -n nativestor-system lvmdconfig-xxxxxx
- 参照正常节点配置将异常节点的status字段修改为ready
