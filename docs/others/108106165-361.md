# 3.6.1

应用发布指定日志路径后运维中心日志查询不到 容器内相关目录下日志显示正常

## Cause
- nevermore组件启动时间过长导致inotify资源耗尽

## Resolution
- 重启nevermore组件临时解决
- 升级到3.7版本修复

## [workaround]
- 重启nevermore组件的pod
