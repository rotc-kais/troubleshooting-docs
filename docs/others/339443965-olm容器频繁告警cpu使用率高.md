# olm容器频繁告警cpu使用率高

olm组件频繁告警，cpu占用率超过90%

## Cause
- 资源配额修改后被还原
- 未正确配置olm-registry-config

## Resolution
- 给pod limit扩成原来的两倍
- 在部署添加skip-sync: "true"的annotation字段
- 修改configmap：kubectl edit cm -n cpaas-system olm-registry-config -oyaml
- 手动修改对应的deploy

## [workaround]
- 临时添加skip-sync: "true"注解防止配置还原
