# databases服务频繁触发重启

服务频繁触发重启 退出代码为137 健康检查失败

## Cause
- 健康检查超时时间设置过短导致探测失败

## Resolution
- 修改StatefulSet的readinessProbe.timeoutSeconds配置：
kubectl edit sts -n命名空间 harbor-databases
将timeoutSeconds从1调整为10
