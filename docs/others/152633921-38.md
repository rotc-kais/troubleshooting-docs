# 3.8

nevermore内存上涨后不释放 内存使用量超过80%

## Cause
- 日志量突增导致nevermore组件buffer达到上限
- 3.8版本nevermore使用spool类型buffer存在缺陷
- 旧版本output采用单goroutine发送数据

## Resolution
- 删除/cpaas/log/data/spool.dat文件
