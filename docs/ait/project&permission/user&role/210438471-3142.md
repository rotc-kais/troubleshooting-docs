# 3.14.2

原生应用使用chart部署失败 绑定项目管理员角色的用户创建应用报错无权限

## Cause
- 用户被加入用户组但未直接分配角色
- 3.14版本存在权限校验bug

## Resolution
- 在global集群的captain-controller-manager-role clusterrole中增加权限配置: 
- apiGroups: 
  - auth.alauda.io
  resources: 
  - users
  verbs: 
  - get
  - list

## [workaround]
