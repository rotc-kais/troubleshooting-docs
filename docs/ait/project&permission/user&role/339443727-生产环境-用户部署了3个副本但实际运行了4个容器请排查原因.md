# 生产环境  用户部署了3个副本，但实际运行了4个容器，请排查原因。

部署3个副本但实际运行4个容器 删除多余容器后自动重新创建

## Cause
- TKE控制台与DevOps流水线双重部署导致配置冲突
- 副本数被不同部署方式多次修改

## Resolution
- 通过统一部署方式重新部署应用
- 确保TKE控制台和DevOps流水线配置一致性
