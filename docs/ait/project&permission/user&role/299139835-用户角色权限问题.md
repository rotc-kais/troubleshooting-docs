# 用户角色权限问题

管理员权限和对应命名空间权限无法查看eplat-dev命名空间的内容

## Cause
- 用户没有被正确绑定到eplat-dev命名空间的访问角色

## Resolution
- 创建或更新RoleBinding，将用户账户绑定到具有eplat-dev命名空间访问权限的角色
- 应用新的RBAC配置后验证访问
- 使用kubectl auth can-i命令验证权限
- 参考Kubernetes RBAC授权指南
