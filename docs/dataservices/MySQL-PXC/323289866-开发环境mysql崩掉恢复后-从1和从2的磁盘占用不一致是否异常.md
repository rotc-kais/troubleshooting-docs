# 开发环境mysql崩掉恢复后 从1和从2的磁盘占用不一致，是否异常

从1和从2的磁盘占用不一致

## Cause
- 崩溃恢复时触发数据clone操作导致日志文件(Binlog/redo log等)重新生成

## Resolution
- 检查MGR集群状态是否正常，确认memberState显示为"OFFLINE"
